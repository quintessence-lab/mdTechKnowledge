## 流れのまとめ

```
記事編集 → git push → GitHub Actions が自動ビルド → GitHub Pages に配信 → 公開URL で閲覧
```

記事を追加したい場合は、`src/content/blog/` に `.md` ファイルを追加して `git push` するだけで、自動的にサイトが更新されます。

## 主要ファイルの役割

| ファイル | 役割 |
|---------|------|
| `astro.config.mjs` | サイトURL・base path・ビルド設定 |
| `.github/workflows/deploy.yml` | 自動ビルド・デプロイの定義 |
| `src/content/config.ts` | 記事のフロントマタースキーマ定義 |
| `src/content/blog/*.md` | 記事本体（Markdown） |
| `src/pages/` | ルーティング・ページテンプレート |
| `src/layouts/` | 共通レイアウト |
| `src/components/` | 再利用可能なUIパーツ |

## GitHub Pages の設定手順

1. GitHub リポジトリの **Settings** > **Pages** を開く
2. **Source** を「GitHub Actions」に変更
3. `main` ブランチへ push すると自動でビルド・デプロイが実行される
4. **Actions** タブでビルド状況を確認できる
