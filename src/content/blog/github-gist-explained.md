# GitHub Gist 完全ガイド — Pages・Actionsとの関係を図解

> 2026年5月時点

「GitHub Pages はサイトを公開するもの、GitHub Actions は自動化するもの、これは分かる。でも Gist って何？ どう使い分ける？」 — Gist は GitHub の中でも目立たない機能ですが、**ブログ・noteへの表埋め込み・スニペット共有・データ公開** など、知っていると役立つ場面が意外と多くあります。本記事では Gist の仕組みから、Pages / Actions / 通常リポジトリとの関係までを図解で整理します。

---

## 1. GitHub Gist とは何か

**ひとことで言うと**: 「**1ファイル〜数ファイルだけ**を共有・公開するための、超軽量な GitHub リポジトリ」です。

```
┌─────────────────────────────────────────────────────────────┐
│  GitHub のサービス階層                                       │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  【リポジトリ】 ─── プロジェクト規模 ────────────────────    │
│      複数フォルダ・大量ファイル・Actions・Pages・Issues       │
│      例: mdTechKnowledge, mdtechknowledge-handoff             │
│                                                              │
│  【Gist】 ─── スニペット規模 ─────────────────────────────   │
│      1〜数ファイル・履歴管理あり・コメント可・埋め込み可      │
│      例: 表データのCSV/Markdown、便利スクリプト1個            │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### Gist と通常リポジトリの違い

| 項目 | 通常のリポジトリ | Gist |
|------|----------------|------|
| **目的** | プロジェクト全体 | 単発のコード片・データ片 |
| **作成手間** | リポジトリ作成→push | テキスト貼り付けて Create |
| **ファイル数** | 多数 | 数個（実用上1〜10個） |
| **フォルダ構造** | あり | なし（フラット） |
| **公開設定** | public / private | public / secret（後述） |
| **履歴管理** | あり（Git） | あり（Git） |
| **埋め込み** | 不可 | 可能（noteやBlogに） |
| **アクセスURL** | `github.com/<user>/<repo>` | `gist.github.com/<user>/<hash>` |

## 2. Gist の3つの状態

```
┌──────────────────────────────────────────────────────────────┐
│                                                               │
│   Public Gist           Secret Gist           Private (なし)   │
│   ───────────           ───────────           ──────────────   │
│   検索可能              URL を知らないと到達不可               │
│   プロフィールに表示     プロフィール非表示    ※ Gistには       │
│   GitHubの検索でhit     リンク共有時のみ閲覧 「完全な非公開」   │
│                         可                    モードはない      │
│                                                                │
└──────────────────────────────────────────────────────────────┘
```

> ⚠️ **重要**: Secret Gist は「URL を知っている人なら誰でも見られる」ので、本物の機密情報には使えません。

## 3. 何ができるのか（用途）

### 用途別の整理

| 用途 | 具体例 |
|------|--------|
| **コード片の共有** | エラー再現の最小コード、設定ファイルのサンプル |
| **データの共有** | CSV/JSON/TSVをそのまま置く（GitHubが自動で表として描画） |
| **記事への埋め込み** | note / dev.to / Qiita / Medium で表や図を貼る |
| **下書き・メモ** | 公開しない Secret Gist として個人メモ |
| **他人のGistをFork** | 改変版を作って公開 |
| **コメント・議論** | Gistにもコメント機能あり |
| **APIから動的取得** | プログラムが Gist API でテンプレ取得 |

### 技術ブログ運用での活用例

```

┌─────────────────────────────────────────────────────────────────┐
```

```
│                                                                  │
```

## 5. Gist と各サービスの関係

### Gist と GitHub Pages

```
┌─────────────────────────────────────────────────────────┐
│  関係性: 独立した別サービス                              │
│                                                          │
│  GitHub Pages       ←(任意の参照)→       Gist            │
│  HTMLサイトを公開                       コード片を公開    │
│  (ブログ・ポートフォリオ)               (スニペット)      │
│                                                          │
│  併用パターン:                                            │
│  ┌────────────────────────────────────────────────┐    │
│  │ Pages サイトの HTML から:                       │    │
│  │   <script src="https://gist.github.com/.../    │    │
│  │              <hash>.js"></script>               │    │
│  │ で Gist の中身を埋め込み可能                    │    │
│  └────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────┘
```

### Gist と GitHub Actions

```
┌─────────────────────────────────────────────────────────┐
│  関係性: 独立しているが、Actions から Gist 操作可能     │
│                                                          │
│  GitHub Actions    ←(API操作)→     Gist                  │
│  自動化ワークフロー               コード片の保管庫        │
│                                                          │
│  併用パターン:                                            │
│  ┌────────────────────────────────────────────────┐    │
│  │ Actions が定期実行で:                           │    │
│  │   1. リポジトリの最新CSVを生成                  │    │
│  │   2. gh gist edit <id> で更新                  │    │
│  │   → Gist が常に最新データを反映                 │    │
│  │                                                  │    │
│  │  例: ステータスバッジの更新、データダッシュ     │    │
│  └────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────┘
```

### Gist と通常リポジトリ

```
┌─────────────────────────────────────────────────────────┐
│  関係性: どちらもGitリポジトリだが粒度が違う              │
│                                                          │
│  ┌─────────────────────┐       ┌─────────────────────┐  │
│  │ Repository           │       │ Gist                 │  │
│  │ ──────────           │       │ ──────                │  │
│  │ 重量級               │       │ 軽量                 │  │
│  │ プロジェクト管理     │       │ スニペット管理       │  │
│  │ Issues/PR/Wiki       │       │ コメントのみ         │  │
│  │ Pages/Actions        │       │ なし                 │  │
│  │ 多人数コラボ         │       │ 個人共有寄り         │  │
│  └─────────────────────┘       └─────────────────────┘  │
│                                                          │
│  Gist もそれぞれが Git リポジトリなので…                 │
│  → ローカルに clone して編集も可能:                      │
│     git clone https://gist.github.com/user/<hash>.git    │
└─────────────────────────────────────────────────────────┘
```

## 6. Gist の作成・操作方法

### 方法① ブラウザから作成（最も簡単）

```
1. https://gist.github.com にアクセス
2. ファイル名を入力（例: data.csv）
3. 中身を貼り付け
4. 右下「Create public gist」または「Create secret gist」
5. URL が発行される
```

### 方法② gh CLI から作成

```bash
gh gist create file.csv --public --desc "説明文"
gh gist create file.md --secret             # secret gist
gh gist list                                # 自分のGist一覧
gh gist view <hash>                         # 中身表示
gh gist edit <hash>                         # ローカルエディタで編集
gh gist delete <hash>                       # 削除
```

### 方法③ API から作成

```bash
curl -X POST https://api.github.com/gists \
  -H "Authorization: token $GITHUB_TOKEN" \
  -d '{
    "description": "...",
    "public": true,
    "files": {"data.csv": {"content": "..."}}
  }'
```

## 7. Gist 埋め込みの仕組み

```
┌───────────────────────────────────────────────────────────────┐
│                                                                │
│  Gist URL                                                      │
│   https://gist.github.com/user/<hash>                          │
│                  │                                             │
│         ┌────────┴─────────┐                                   │
│         ▼                  ▼                                   │
│  通常閲覧            JavaScript埋め込み                        │
│  (ブラウザでGist          .js を読み込ませる                   │
│   ページを表示)                                                │
│                       URL末尾に .js を付けると                 │
│                       <script> 用の埋め込みコードを返却        │
│                                                                │
│  実装例:                                                        │
│   <script src="https://gist.github.com/user/<hash>.js">        │
│   </script>                                                    │
│                                                                │
│  noteの場合:                                                    │
│   noteエディタで「埋め込み」→ Gist URLを貼るだけ                │
│   noteが内部で .js 埋め込みを生成                              │
│                                                                │
└───────────────────────────────────────────────────────────────┘
```

### 埋め込みの見え方

| 形式 | noteでの表示 | 利点 |
|------|------------|------|
| **CSV** | ソート可能・検索可能のグリッド表 | データ閲覧に便利 |
| **MD** | GitHubの整形済み表（ヘッダー強調） | 見た目が綺麗 |
| **コード（py/js等）** | シンタックスハイライト付き | 行番号・色付け |
| **JSON** | 折りたたみ可能ツリー | 大きな構造化データ |

## 8. メリットとデメリット

### メリット

| 項目 | 内容 |
|------|------|
| **無料** | GitHubアカウントがあれば無制限に作成可能 |
| **CDN高速配信** | GitHub のインフラで配信される |
| **Git履歴あり** | 誰がいつ何を変更したか追跡可能 |
| **埋め込み標準対応** | note・dev.to・Qiita・はてなブログ等が対応 |
| **API豊富** | プログラムからの自動操作が容易 |

### デメリット・注意点

| 項目 | 内容 |
|------|------|
| **完全な非公開不可** | Secret でもURL知る人は閲覧可 |
| **大きなファイル不向き** | 単発スニペット用、数MB級は避ける |
| **検索性弱い** | publicでも階層がないため探しにくい |
| **GitHub依存** | アカウント停止リスク |
| **自分のアカウントに紐付く** | 大量作成すると一覧が膨大に |

## 9. 使い分けの指針

### 「Gist にすべきか、リポジトリにすべきか」の判断基準

| 状況 | 推奨 |
|------|------|
| 1〜数ファイル、サクッと共有したい | **Gist** |
| プロジェクトとして継続的に育てる | **Repository** |
| ブログ・noteに表を埋め込みたい | **Gist** |
| 自動デプロイで HTMLサイトを公開 | **Repository + Pages + Actions** |
| 機密情報を保管 | **どちらも不可**（Secret Gistも他人に閲覧され得る） |
| データだけ提供したい | **Gist (CSV/JSON)** |
| Issue や PR で議論したい | **Repository** |

## 10. まとめ

```
Gist = 「軽量なリポジトリ」
       Pages（サイトホスティング）とも、
       Actions（CI/CD）とも、
       通常リポジトリとも、
       独立した別サービス。

       でも、URL ベースで連携させて、
       様々な場面で「コード片・データ片の共有窓口」として使える。
```

- **Repository** はプロジェクト規模、**Gist** はスニペット規模
- **Pages** と **Actions** は Repository に紐付くサービス、**Gist** とは独立
- **Public** は検索可能、**Secret** は URL 共有のみ（完全非公開ではない）
- 埋め込みは **`.js` を付与した URL** または各プラットフォームの埋め込み機能で実現
- ブログ・note等で **表を綺麗に表示する用途** で特に有用

技術ブログを書く・コードを共有する・データを公開する場面で、Gist は「リポジトリほど重くなく、ローカルファイル共有ほど一過性でもない」中間的な選択肢として機能します。仕組みを理解しておくと、外部プラットフォームへの転載や読者への補助資料提供で活用の幅が広がります。
