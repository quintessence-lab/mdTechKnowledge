## Claude Code Routines とは

2026年4月14日にリサーチプレビューとしてリリースされた、Claude Codeのクラウド自動化機能です。

**プロンプト + GitHubリポジトリ + MCPコネクタ**（Slack、Linear、Google Driveなど）をセットで登録し、定期実行やイベント駆動でClaude Codeセッションを自動実行できます。

---

## 実行環境

### どこで動くのか？

Routinesは **Anthropic管理のクラウドインフラ上** で実行されます。ローカルPCは不要で、PCがオフでも動作します。

| 項目 | 詳細 |
|:-----|:-----|
| 実行場所 | Anthropicクラウド上のサンドボックス |
| OS | Linux（bubblewrapによるサンドボックス） |
| ローカルPC | 不要（PCを閉じていても実行される） |
| セッション形式 | 実行ごとにフルのClaude Codeクラウドセッションが起動 |

### リポジトリとの連携

- 実行ごとにGitHubリポジトリが **フルクローン** される（デフォルトブランチから）
- デフォルトでは `claude/` プレフィックスのブランチのみpush可能
- 「Allow unrestricted branch pushes」を有効にすれば全ブランチにpush可能
- read-onlyではなく、コードの変更・コミット・pushが可能

### セキュリティ・サンドボックス

- **ファイルシステム隔離**: 作業ディレクトリ（クローンされたリポジトリ）への読み書きのみ許可。外部ファイルへの書き込みはブロック
- **ネットワーク隔離**: プロキシ経由で許可ドメインのみ接続可能
- **認証情報の保護**: Gitクレデンシャル等の機密情報はサンドボックス内に入らない
- **自律実行**: 承認プロンプトなしで動作するが、スコープは設定で制限

### MCPコネクタ

claude.aiで接続済みのMCPコネクタ（Slack、Linear、Google Driveなど）をルーティンに含めることで、実行時に外部サービスへの読み書きが可能です。不要なコネクタは除外できます。

### 環境設定

Environment設定で以下をカスタマイズ可能です。

- ネットワークアクセスレベル
- 環境変数
- セットアップスクリプト（依存関係インストール等）

## トリガー（実行条件）

Routinesには3種類のトリガーがあり、用途に応じて使い分けます。

| トリガー種別 | 起動条件 | 実行タイミング | イベントがない日 |
|:------------|:---------|:-------------|:---------------|
| スケジュール（定期実行） | 時刻（cron式） | 固定スケジュール | それでも実行される |
| GitHub（イベント駆動） | GitHubイベント | イベント発生時 | 実行されない |
| API | HTTPリクエスト | 任意 | 実行されない |

### スケジュールトリガー（定期実行）

cron式で指定した時刻に、Anthropicクラウドが自動的にRoutineを起動します。

- 何もイベントが起きていなくても、定刻に毎回実行される
- 実行間隔は最短1時間（時間単位 / 毎日 / 平日 / 週1）
- 例: 毎朝7時にウェブから最新情報を収集してレポート生成

```
時計ベースで自動起動（イベント有無に関係なく実行）

 ┌─────┐    ┌─────┐    ┌─────┐
 │ Day1│    │ Day2│    │ Day3│  ...
 │ 7:00│    │ 7:00│    │ 7:00│
 └──┬──┘    └──┬──┘    └──┬──┘
    ▼          ▼          ▼
 Routine    Routine    Routine
 実行        実行        実行
```

### GitHubトリガー（イベント駆動）

GitHubリポジトリにwebhookが登録され、特定のイベント（push、PR作成、Issueコメントなど）が発生した瞬間にRoutineが起動します。

- イベントがなければ一切実行されない（無駄なコスト消費なし）
- リアルタイムに反応できる
- 例: PRが作成されたら自動コードレビュー、pushされたらテスト実行

```
GitHubリポジトリでイベント発生時のみ起動

 git push ──────┐
 PR作成 ────────┤
 Issue作成 ─────┼──▶ webhook ──▶ Routine実行
 PR comment ────┤
 workflow完了 ──┘
```

### APIトリガー

Routine作成時に専用のHTTPエンドポイントが発行され、外部からHTTP POSTリクエストで任意のタイミングに起動できます。

- bearerトークンで認証
- 外部システム、スクリプト、他のRoutineから呼び出し可能
- 例: CI/CDパイプラインの一部として実行、手動テスト実行

## 料金

Routinesはサブスクリプションに含まれており、追加費用はかかりません。

| プラン | 月額 | Routines上限 |
|:------|:-----|:------------|
| Pro | $20 | 5回/日 |
| Max 5x | $100 | 15回/日 |
| Max 20x | $200 | 15回/日 |
| Team / Enterprise | — | 25回/日 |

週1回程度の実行であれば、どのプランでも十分余裕があります。

### 公式日次実行上限（スケジュール制限）

Routines の **スケジュールトリガーは、プランごとに 1 日あたりの自動実行回数に上限**があります。GitHub トリガーや API トリガーによる起動とは別枠で、`Routine × 起動回数/日` の合計でカウントされます。

| プラン | スケジュール起動上限 |
|:------|:------|
| Pro | 5 回/日 |
| Max（5x / 20x 共通） | 15 回/日 |
| Team / Enterprise | 25 回/日 |

- 上限はアカウント単位の **合計実行回数**（複数 Routine を登録していても合算）
- 上限に達した日のスケジュール起動はスキップされ、翌日にリセットされる
- イベント駆動（GitHub Webhook）や API 起動は本上限の対象外（ただしサブスクリプションの利用枠は通常通り消費）

スケジュール頻度を高く設定すると上限を消費しやすいため、毎時実行のような構成は Max 以上が現実的です。

### 【重要】2026年6月15日のプログラマティック利用 課金分離（要確認）

Anthropic は 2026年6月15日付で、**プログラマティック（自動・無人）利用をサブスクリプション枠から切り離し、独立した月次クレジットプールから消費させる** 課金変更を実施します（発表は2026年5月14日、対象者には6/15前にクレジット申請の案内メールが送付されます）。

公式ヘルプが明示している **対象** は以下の4種です。

- Claude Agent SDK（Python / TypeScript）
- `claude -p`（非対話・ヘッドレスモード）
- Claude Code GitHub Actions
- Agent SDK で認証する3rdパーティアプリ

一方、**claude.ai のWeb/デスクトップ/モバイルチャット、ターミナルでの対話的 Claude Code、Claude Cowork は従来どおりサブスク枠** を消費し、本変更の対象外とされています。

| 項目 | 内容 |
|:-----|:-----|
| 発効日 | 2026年6月15日 |
| 月次クレジット額 | Pro $20 / Max 5x $100 / Max 20x $200（Team Standard $20・Premium $100/席） |
| 課金レート | 全額 API 料金（full API rates） |
| 繰越 | なし（毎月リフレッシュ） |
| 枯渇時 | usage credits を有効化していれば API 料金で継続課金／無効なら次サイクルまでリクエスト停止（capped） |

**Routines への影響（推定・要確認）**: Routines は Anthropic クラウド上で **人間の監視なしに自律実行される無人セッション** であり、実務上は「各ターンを人間が監視せずに動くセッションは新クレジットプールへ移行する」という整理に該当する公算が高いと考えられます。このため、**6月15日以降、Routines の実行（とりわけ API トリガー等のプログラマティックな起動）が独立クレジットプールから消費される可能性** があります。

ただし、現時点（2026年6月初旬）で確認できる公式ヘルプ・各報道のいずれも、対象として **「Routines」を名指しで列挙していません**。したがって本項は推定であり、運用前に **公式の最終確認を推奨** します。クレジットプールが枯渇した場合、usage credits を無効化していると **Routines が次の請求サイクルまで自動停止（capped）する** 点にも注意が必要です。

## 活用例

- **定期レポート生成**: 毎週月曜にリポジトリの変更サマリーをSlackに投稿
- **PRの自動レビュー**: PRが作成されたらコードレビューを実行しコメント
- **issueのトリアージ**: 新しいissueにラベルを自動付与
- **ドキュメント更新チェック**: コード変更に伴うドキュメントの整合性を定期チェック

## Desktopアプリ統合（2026-04-15リリース）

Routinesは2026年4月15日のClaude Code **デスクトップアプリ大規模リデザイン** と同時にリリースされ、UI上で密接に統合されています。

### リデザインされたDesktopアプリからの利用

- 刷新されたClaude Code Desktopアプリ（macOS / Windows）から、Routinesの作成・編集・実行・履歴閲覧をGUIで操作可能
- ターミナルに加え、Desktopアプリがクラウド機能（Routines含む）の主要なエントリポイントに位置づけられた
- Routine毎の実行ログ・成果物（PR、コミット、Slack投稿など）をアプリ内で確認できる

### 並列セッション管理との連携

リデザインで導入された **マルチセッションサイドバー** から、複数のRoutinesを同時に起動・監視できます。

- 複数のRoutineを並列で実行し、サイドバーから進捗をリアルタイムに切り替え表示
- ローカルセッションとクラウドRoutineを同一UIで横断的に管理
- 長時間動作するRoutineをバックグラウンドで走らせつつ、別セッションで対話的な作業を継続可能

これにより、Routinesは「裏側で勝手に動くスケジューラ」だけでなく、「ユーザーが能動的にオーケストレーションする並列ワーカー」としても扱えるようになりました。

## `/goal` コマンドと Agent View — 無人実行の補強機能（v2.1.139〜）

2026年5月11日（PT）/ 5月12日（JST）リリースの **Claude Code v2.1.139** で、Routinesと同じ「自律実行」の文脈に直結する2つの機能が追加されました。

### `/goal` コマンド — 完了条件達成までの自動継続

`/goal <completion condition>` 形式で **完了条件を自然言語で指定** すると、Claude が条件を満たすまで複数ターンにわたり自律的にタスクを継続実行します。

| 観点 | 仕様 |
|:-----|:-----|
| 起動形式 | `/goal <完了条件>` （例: `/goal すべてのテストが通るまで修正を続けて`） |
| 動作モード | インタラクティブ / `-p`（非対話） / Remote Control |
| UIオーバーレイ | 経過時間・ターン数・トークン消費量をリアルタイム表示 |
| キャンセル | Esc キー（v2.1.140〜） |
| 実行制御 | サブエージェント / ツール実行中は次ターン発火を抑止（v2.1.141修正） |
| Stop/SubagentStop hooks | `/goal` 評価中は発火しない（design） |

**Routines との関係**: Routines がスケジュール/イベント駆動で「**いつ動かすか**」を定義するのに対し、`/goal` は1セッション内で「**どこまで動くか**」を定義します。両者を組み合わせれば、「毎朝のRoutineが起動 → セッション内で `/goal` が完了条件まで自動継続」という二段構えの無人実行が可能です。

### `claude agents` Agent View — 全セッション統合管理画面

`claude agents` コマンドで、ローカル・背景・Remote含む **すべての Claude Code セッションを1画面で管理** できるダッシュボードが起動します（リサーチプレビュー）。

| ステータス | 内容 |
|:-----|:-----|
| **Running** | アクティブ実行中 |
| **Awaiting Input** | ユーザー応答待ち |
| **Completed** | 完了済（経過時間 `3h 2m 5s` 形式で表示） |
| **Background (`bg`)** | 背景実行中（ピン留めでアイドル時生存） |
| **Stopped** | ユーザー停止済 |

**主要キーバインド**:

| キー | 機能 |
|:-----|:-----|
| `→` / `←` | セッションへの attach / detach |
| `Ctrl+T` | 背景セッション新規作成 |
| `Ctrl+R` | セッションリネーム |
| `v` | エディタで開く（`$EDITOR` / `$VISUAL`） |
| `x` | サブエージェント停止 |
| `d` | セッション削除 |

**Routines との位置づけ**: クラウドで動くRoutineと、ローカル / 背景セッションが Agent View 上で **横断的に可視化** されます。複数の自動化ストリームが並走する際に「どれが今走っているか」「どれが入力待ちか」を一覧できるため、無人実行を多重化したワークフローの統制に有用です。

### v2.1.142での拡張（2026-05-14 PT / 5-15 JST）

v2.1.142 では Agent View に CLI フラグが拡張され、ダッシュボード経由で起動するセッションへ以下を継承可能になりました:

```bash
claude agents \
  --add-dir <path> \
  --settings <file> \
  --mcp-config <file> \
  --permission-mode <mode> \
  --model <model> \
  --effort <level> \
  --dangerously-skip-permissions
```

これにより、Routines の API トリガー経由で「特定の permission-mode・model・effort で背景セッションを起動」というパターンが Agent View からも統一的に扱えるようになりました。

## Routines と /ultrareview の位置づけ比較

2026年4月22日（v2.1.111）にリリースされた **`/ultrareview`** は、Routinesと同じく「クラウドで動くClaude Code」という共通基盤を持ちながら、異なる役割を担います。

| 観点 | Routines | /ultrareview |
|:-----|:---------|:------------|
| 目的 | 繰り返しタスクのスケジュール自動化 | PRやブランチの深掘りコードレビュー |
| 起動方法 | スケジュール / GitHubイベント / API（自動） | `/ultrareview` コマンドまたは `claude ultrareview` CLI（手動） |
| 実行場所 | Anthropicクラウドのサンドボックス | Anthropicクラウドのサンドボックス（同基盤） |
| エージェント数 | 1セッション（直列処理） | 複数レビュアーエージェントが並列で独立検証 |
| 所要時間 | タスク内容により様々 | 5〜10分程度 |
| 主なユースケース | 情報収集・記事更新・Issue自動処理 | マージ前の徹底バグハント・独立検証 |
| 料金 | サブスクリプション内（追加費用なし） | 全プラン共通で $5〜$20/回（Extra usage、2026-05-05 にPro/Maxの無料ラン3回は失効） |
| CI連携 | APIトリガーで任意タイミングに呼び出し可 | `claude ultrareview` サブコマンドでCI組み込み可 |

### 使い分けの指針

- **Routines**: 「定期的・自動的にやること」を登録しておく仕組み。情報収集・記事更新・Issueトリアージなどの反復業務に向く。
- **/ultrareview**: 「このPRを本当にマージして大丈夫か」という1回ごとの深掘り検証に向く。ローカルで動く `/review`（単一パス・秒〜数分・追加費用なし）と使い分けることで費用対効果を最大化できる。
- **組み合わせ**: RoutinesのAPIトリガーを活用すれば、PR作成後に自動で `claude ultrareview` を呼び出すCI/CDフローの構築も可能。
