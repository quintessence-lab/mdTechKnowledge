## Claude Code Routines とは

2026年4月14日にリサーチプレビューとしてリリースされた、Claude Codeのクラウド自動化機能です。

**プロンプト + GitHubリポジトリ + MCPコネクタ**（Slack、Linear、Google Driveなど）をセットで登録し、定期実行やイベント駆動でClaude Codeセッションを自動実行できます。

---

## 実行環境

### どこで動くのか？

Routinesは **Anthropic管理のクラウドインフラ上** で実行されます。ローカルPCは不要で、PCがオフでも動作します。

| 項目 | 詳細 |
|:-----|:-----|
| 実行場所 | Anthropicクラウド上のサンドボックス |
| OS | Linux（bubblewrapによるサンドボックス） |
| ローカルPC | 不要（PCを閉じていても実行される） |
| セッション形式 | 実行ごとにフルのClaude Codeクラウドセッションが起動 |

### リポジトリとの連携

- 実行ごとにGitHubリポジトリが **フルクローン** される（デフォルトブランチから）
- デフォルトでは `claude/` プレフィックスのブランチのみpush可能
- 「Allow unrestricted branch pushes」を有効にすれば全ブランチにpush可能
- read-onlyではなく、コードの変更・コミット・pushが可能

### セキュリティ・サンドボックス

- **ファイルシステム隔離**: 作業ディレクトリ（クローンされたリポジトリ）への読み書きのみ許可。外部ファイルへの書き込みはブロック
- **ネットワーク隔離**: プロキシ経由で許可ドメインのみ接続可能
- **認証情報の保護**: Gitクレデンシャル等の機密情報はサンドボックス内に入らない
- **自律実行**: 承認プロンプトなしで動作するが、スコープは設定で制限

### MCPコネクタ

claude.aiで接続済みのMCPコネクタ（Slack、Linear、Google Driveなど）をルーティンに含めることで、実行時に外部サービスへの読み書きが可能です。不要なコネクタは除外できます。

### 環境設定

Environment設定で以下をカスタマイズ可能です。

- ネットワークアクセスレベル
- 環境変数
- セットアップスクリプト（依存関係インストール等）

## トリガー（実行条件）

Routinesには3種類のトリガーがあり、用途に応じて使い分けます。

| トリガー種別 | 起動条件 | 実行タイミング | イベントがない日 |
|:------------|:---------|:-------------|:---------------|
| スケジュール（定期実行） | 時刻（cron式） | 固定スケジュール | それでも実行される |
| GitHub（イベント駆動） | GitHubイベント | イベント発生時 | 実行されない |
| API | HTTPリクエスト | 任意 | 実行されない |

### スケジュールトリガー（定期実行）

cron式で指定した時刻に、Anthropicクラウドが自動的にRoutineを起動します。

- 何もイベントが起きていなくても、定刻に毎回実行される
- 実行間隔は最短1時間（時間単位 / 毎日 / 平日 / 週1）
- 例: 毎朝7時にウェブから最新情報を収集してレポート生成

```
時計ベースで自動起動（イベント有無に関係なく実行）

 ┌─────┐    ┌─────┐    ┌─────┐
 │ Day1│    │ Day2│    │ Day3│  ...
 │ 7:00│    │ 7:00│    │ 7:00│
 └──┬──┘    └──┬──┘    └──┬──┘
    ▼          ▼          ▼
 Routine    Routine    Routine
 実行        実行        実行
```

### GitHubトリガー（イベント駆動）

GitHubリポジトリにwebhookが登録され、特定のイベント（push、PR作成、Issueコメントなど）が発生した瞬間にRoutineが起動します。

- イベントがなければ一切実行されない（無駄なコスト消費なし）
- リアルタイムに反応できる
- 例: PRが作成されたら自動コードレビュー、pushされたらテスト実行

```
GitHubリポジトリでイベント発生時のみ起動

 git push ──────┐
 PR作成 ────────┤
 Issue作成 ─────┼──▶ webhook ──▶ Routine実行
 PR comment ────┤
 workflow完了 ──┘
```

### APIトリガー

Routine作成時に専用のHTTPエンドポイントが発行され、外部からHTTP POSTリクエストで任意のタイミングに起動できます。

- bearerトークンで認証
- 外部システム、スクリプト、他のRoutineから呼び出し可能
- 例: CI/CDパイプラインの一部として実行、手動テスト実行

## 料金

Routinesはサブスクリプションに含まれており、追加費用はかかりません。

| プラン | 月額 | Routines上限 |
|:------|:-----|:------------|
| Pro | $20 | 5回/日 |
| Max 5x | $100 | 15回/日 |
| Max 20x | $200 | 15回/日 |
| Team / Enterprise | — | 25回/日 |

週1回程度の実行であれば、どのプランでも十分余裕があります。

---

## 活用例

- **定期レポート生成**: 毎週月曜にリポジトリの変更サマリーをSlackに投稿
- **PRの自動レビュー**: PRが作成されたらコードレビューを実行しコメント
- **issueのトリアージ**: 新しいissueにラベルを自動付与
- **ドキュメント更新チェック**: コード変更に伴うドキュメントの整合性を定期チェック
