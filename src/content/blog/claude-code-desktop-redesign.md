**最終更新**: 2026-04-26

2026年4月15日、Anthropicは Claude Code Desktopアプリ（macOS / Windows、`code.claude.com`）を **IDE型ワークスペース** として全面リデザインした。本記事では、リデザインで追加された主要機能と従来のCLI版との違いを整理する。

> **注**: クラウドセッションの通信アーキテクチャ（端末 → Anthropicクラウド → サンドボックスコンテナ → GitHub）は変更なし。本リデザインは **Desktop版のUI** に対するものであり、CLI版（`claude` コマンド）は対象外。

## 全体構成

リデザイン後のDesktopアプリは、左サイドバーと自由配置可能なメインエリアの2カラム構成。VS CodeのようなIDE体験をクラウドネイティブのClaude Code上で提供する。

```
┌───────────────────────────────────────────────────────────────┐
│  Claude Code Desktop                                          │
│                                                               │
│  ┌──────────┐ ┌──────────────────────────────────────────┐    │
│  │サイドバー│ │  メインエリア（ドラッグ&ドロップ可能）   │    │
│  │          │ │                                          │    │
│  │ セッション │ │  ┌──────────┐ ┌──────────┐              │    │
│  │ 一覧     │ │  │ チャット │ │ Diff     │              │    │
│  │          │ │  │          │ │ ビューア │              │    │
│  │ Routines │ │  └──────────┘ └──────────┘              │    │
│  │          │ │  ┌──────────┐ ┌──────────┐              │    │
│  │ ステータス│ │  │ 統合     │ │ プレビュー│             │    │
│  │ フィルタ │ │  │ ターミナル│ │ ペイン   │              │    │
│  │          │ │  └──────────┘ └──────────┘              │    │
│  │ サイド   │ │                                          │    │
│  │ チャット │ │                                          │    │
│  │ (Cmd+;)  │ │                                          │    │
│  └──────────┘ └──────────────────────────────────────────┘    │
└───────────────────────────────────────────────────────────────┘
         │ HTTPS (WebSocket)
         ▼
  Anthropicクラウド → サンドボックスコンテナ → GitHub
  （通信アーキテクチャは従来と同一）
```

## 主な新機能

| 機能 | 説明 |
|------|------|
| **マルチセッションサイドバー** | アクティブ・最近のセッションを一覧管理。複数リポジトリを並行操作可能。クラウド対話セッションとRoutinesを横断管理 |
| **ドラッグ&ドロップレイアウト** | チャット・Diff・ターミナル・プレビューを自由に配置・分割。作業内容に合わせて任意に組み替え可能 |
| **統合ターミナル** | セッションの作業ディレクトリで直接コマンド実行。AIへの指示と手動コマンドの混在ワークフローに対応 |
| **再構築Diffビューア** | 大規模変更セットのレビュー時もスムーズに動作するよう内部実装を刷新 |
| **ファイルエディタ** | チャットやDiffに表示されたファイルパスをクリックして直接編集。簡易な手修正が可能 |
| **プレビューペイン** | HTML・PDF・画像をアプリ内でプレビュー表示。Web成果物の確認をシームレスに |
| **サイドチャット**（Cmd+;） | 実行中タスクのスレッドに影響を与えず、別チャットで質問を分岐できる |
| **3つの表示モード** | Verbose / Normal / Summary でツール呼び出しの詳細度を切替可能 |
| **Routines統合** | 同日リリースのRoutinesをGUI操作可能。スケジュール設定・実行ログ・成果物確認をアプリ内で完結 |

## マルチセッションサイドバー

リデザインの最重要機能。従来のClaude Codeは「1セッション ＝ 1ターミナル」が基本だったが、Desktop版では複数セッションを並列に管理できる。

### 表示構造

```
サイドバー
├─ 🟢 Active (実行中)
│   ├─ セッションA  (repo-x branch:feature)
│   ├─ セッションB  (repo-y branch:bugfix)
│   └─ Routine: 毎日のPRトリアージ
├─ 💤 Recent (最近)
│   ├─ セッションC  (3時間前)
│   └─ セッションD  (昨日)
└─ ⏰ Scheduled (Routines)
    ├─ 週次レポート生成
    └─ MCP接続ヘルスチェック
```

### 主なメリット

- **コンテキストスイッチ削減**: 異なるリポジトリ・ブランチでの作業を画面遷移なしに行き来可能
- **長時間タスクの待機が不要**: 重い処理を投げたまま別セッションで作業継続
- **Routinesとの統合管理**: 自動実行ジョブとリアルタイム対話を同じUIで把握

## Routinesとの統合

2026年4月15日にリサーチプレビューが公開された **Routines**（スケジュール / API / GitHubイベント駆動の自動実行機能）も、リデザイン後のDesktopアプリから直接操作できる。

| 操作 | UIからできること |
|------|-----------------|
| Routine作成 | スケジュール（cron） / トリガー条件 / 使用ツール / リポジトリをGUIで設定 |
| 実行ログ確認 | 過去の実行履歴・成果物・エラーをアプリ内で参照 |
| 手動トリガー | スケジュール待たずに即時実行 |
| 一時停止 | 個別Routineを無効化 |

CLI版ではAPI経由での操作のみだった部分が、Desktop版ではGUIで完結する点が大きい。

## 3つの表示モード

リデザインでツール呼び出しの可視性を3段階で切り替えられるようになった。

| モード | 用途 | 表示される情報 |
|--------|------|---------------|
| **Verbose** | デバッグ・学習用途 | 全ツール呼び出し・引数・結果を詳細表示 |
| **Normal**（既定） | 通常作業 | 主要なツール呼び出しと結果のみ表示 |
| **Summary** | 監視・ハイオーバービュー | サマリーとファイル変更のみ。ノイズを最大限削減 |

長時間のタスクをサイドバーに並べて監視する用途では Summary モード、個別セッションで深く作業するときは Verbose、というように使い分け可能。

## CLI版との比較

| 項目 | CLI版（`claude`） | Desktop版（リデザイン後） |
|------|------------------|--------------------------|
| 実行環境 | ローカルターミナル | クラウド（Anthropicサンドボックス） |
| UI | テキスト（Ink/Yoga） | ネイティブGUI |
| マルチセッション | ターミナルマルチプレクサ依存 | サイドバーで標準対応 |
| Diffビューア | テキスト差分 | グラフィカルDiff（再構築済） |
| プレビュー | 不可 | HTML/PDF/画像をアプリ内表示 |
| ファイルエディタ | エディタ呼び出し | 内蔵エディタで直接編集 |
| Routines | API経由のみ | GUIで作成・監視 |
| 対象用途 | ローカル開発・スクリプティング | クラウド主体のチーム開発 |

CLI版は今回のリデザインの **対象外** で、従来通りターミナルで動作する。CLI版自身も継続的に進化しており、Desktop版の代替ではなく **使い分け** の関係にある。

## サイドチャット（Cmd+;）の使いどころ

リデザインで地味に有用な機能。実行中の長時間タスクに割り込まずに、別チャットで質問・調査を行える。

```
[メインチャット]                  [サイドチャット (Cmd+;)]
"このリポジトリ全体に          ↔  "TypeScriptの XX の挙動を
 ESLintを導入して..."             教えて"
 (実行中、長時間)                 (即答、メインに影響なし)
```

メインスレッドのコンテキストを汚さずに済むため、調査・質問専用のスクラッチパッドとして使える。

## まとめ

- **2026年4月15日にClaude Code Desktopアプリが全面リデザイン** ─ IDE型ワークスペースへ刷新
- **マルチセッションサイドバー** が最大の変更点。複数リポジトリ・複数Routinesを横断管理
- **ドラッグ&ドロップレイアウト** で作業内容に合わせたUIカスタマイズが可能
- **Routines統合** によりスケジュール実行ジョブのGUI管理が初めて実現
- **CLI版は対象外**。Desktop版はクラウド主体のチーム開発、CLI版はローカル開発と使い分け
- **通信アーキテクチャは従来と同一** ─ Anthropicクラウド経由のサンドボックスコンテナで実行
