# GitHub の仕組み — ローカルフォルダ・ローカルgit・GitHub の3層構造

> 2026年4月時点

---

## 1. 3層構造の全体像

```
┌─────────────────────────────────────────────────────────────────────┐
│  ① ローカル端末のフォルダ（作業ディレクトリ）                         │
│                                                                     │
│  普通のフォルダ。ファイルを作成・編集・削除する場所。                   │
│  この時点では git は関与していない。                                   │
│                                                                     │
│  C:\Users\<user>\myproject\                                         │
│    ├── src\                                                         │
│    ├── README.md    ← エディタで自由に編集                           │
│    └── index.html   ← 保存しただけでは git に記録されない             │
│                                                                     │
│         │                                                           │
│         │ git add（変更をステージング）                                │
│         │ git commit（変更を記録）                                    │
│         ▼                                                           │
├─────────────────────────────────────────────────────────────────────┤
│  ② ローカル git（.git フォルダ）                                     │
│                                                                     │
│  同じフォルダ内の隠しフォルダ `.git/` に履歴が保存される。              │
│  インターネット接続は不要。PC内で完結する。                            │
│                                                                     │
│  C:\Users\<user>\myproject\.git\                                    │
│    ├── objects\    ← ファイルの全バージョンを圧縮保存                  │
│    ├── refs\       ← ブランチ・タグの情報                             │
│    └── HEAD        ← 現在のブランチ                                  │
│                                                                     │
│  できること:                                                         │
│    git log     → 変更履歴を見る                                      │
│    git diff    → 変更内容を比較                                      │
│    git branch  → ブランチを作る                                      │
│    git checkout → 過去のバージョンに戻る                              │
│                                                                     │
│  ※ ここまではすべてオフラインで動作する                               │
│                                                                     │
│         │                                                           │
│         │ git push（ローカルの履歴をGitHubに送信）                    │
│         │ git pull（GitHubの変更をローカルに取得）                    │
│         ▼                                                           │
├─────────────────────────────────────────────────────────────────────┤
│  ③ GitHub（リモートリポジトリ）                                      │
│                                                                     │
│  インターネット上のサーバー。ローカル git の複製を保管する。            │
│                                                                     │
│  https://github.com/organization/myproject                          │
│                                                                     │
│  できること（ローカル git にはない機能）:                              │
│    - Pull Request   → コードレビューを依頼                           │
│    - Issues         → バグ・タスク管理                               │
│    - GitHub Actions → 自動ビルド・テスト・デプロイ                    │
│    - GitHub Pages   → 静的サイトの公開                               │
│    - アクセス管理    → public / private の切り替え                    │
│    - バックアップ    → PCが壊れてもコードが残る                       │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

---

## 2. コマンドと3層の関係

```
操作                    ①フォルダ    ②ローカルgit    ③GitHub
──────────────────     ─────────    ────────────    ─────────
ファイルを編集            ✓
git add                  ✓ → ✓
git commit                           ✓
git push                             ✓ → → → →       ✓
git pull                 ✓ ← ← ←    ✓ ← ← ← ←       ✓
git clone                ✓ ← ← ←    ✓ ← ← ← ←       ✓
```

---

## 3. 具体例：記事を公開するまでの流れ

```
手順    操作                          層の動き
────    ────                          ────────
 1      記事ファイルを作成・編集        ① フォルダ
 2      git add article.md            ① → ②（ステージング）
 3      git commit -m "記事追加"       ② に記録
 4      git push                      ② → ③（GitHubに送信）
 5      GitHub Actions が自動実行      ③ 内部で npm run build
 6      GitHub Pages に公開            ③ → Web公開
```

---

## 4. git管理しているフォルダとしていないフォルダの違い

```
                    git管理なし              git管理あり
─────────────      ──────────────          ────────────────────
.git フォルダ       なし                     あり
変更履歴            なし                     git log で全履歴参照可能
元に戻す            不可（上書きしたら終わり） git checkout で任意時点に復元
バックアップ        ローカルのみ              GitHub にも保管
他の人と共有        ファイルを手動コピー       git push / pull で同期
```
