## 1. はじめに — ガバメントAIとは何か

「**ガバメントAI**」は、政府・行政機関が自組織の業務に特化して導入・運用する生成AI基盤の総称です。商用 ChatGPT 等の外部サービスを単に職員に使わせるのではなく、

- **機密情報の外部流出を防ぎつつ**
- **行政固有の業務（議事録要約・文書校正・法令検索など）に最適化された UI・データソース**を備え
- **全府省庁レベルでガバナンス（監査・利用統計・モデル統制）を効かせる**

ことを目的とした「組織内 AI ポータル」がその実体です。

そして、その日本における代表例が、**デジタル庁が開発・運用する「源内（GenAI、ゲンナイ）」** です。本シリーズでは、源内が **2026年4月24日に MIT ライセンスで OSS 化** されたことを契機に、その全体像・アーキテクチャ・実装・ユースケースを4回に分けて解説していきます。

## 2. 源内プロジェクトの背景と狙い

### 2-1. なぜ「源内」なのか

「源内」という名前は、江戸時代の発明家・博物学者として知られる **平賀源内** に由来します。一人で多分野（電気、医学、文芸、鉱山）を横断して試行錯誤した姿を「行政職員が AI でさまざまな業務に取り組む」イメージに重ねた命名と紹介されています。

### 2-2. 規模感 — 全府省庁・約18万人対象

源内は **全府省庁・約18万人の政府職員を対象** とした AI 基盤です。中央省庁レベルでこの規模の AI 基盤を**実運用しながら OSS 化**する事例は世界的にも稀で、これが OSS 公開後に大きな反響（公開3週間で genai-web が 465 stars、genai-ai-api が 288 stars）を呼んでいる理由でもあります。

### 2-3. OSS 化の狙い

| 狙い | 内容 |
|---|---|
| **自治体への横展開** | 47都道府県・1,718市町村への展開を、各自治体がフォークして独自運用する形で支援 |
| **産業界との協働** | 大企業の内製AI基盤の参考実装として活用してもらう |
| **透明性の確保** | 政府が使う AI がどう作られているかを公開することで、市民の信頼を確保 |
| **エコシステム形成** | ExApp（後述）プロトコルを公開することで、AI アプリ提供事業者の市場形成を促す |

## 3. 公開リポジトリ — 2系統の役割分担

源内は GitHub の `digital-go-jp` 組織下に **2つのリポジトリ** で公開されています。

| リポジトリ | 役割 | スター | フォーク |
|---|---|---|---|
| [`digital-go-jp/genai-web`](https://github.com/digital-go-jp/genai-web) | **源内Web** — 利用者が直接触る Web アプリ（インターフェース層） | ⭐465 | 🍴30 |
| [`digital-go-jp/genai-ai-api`](https://github.com/digital-go-jp/genai-ai-api) | **源内AIアプリ** — 行政実務用 AI マイクロサービス群 | ⭐288 | 🍴26 |

> ※ 数値は 2026年5月9日時点のもの。

この2層構成により、**インターフェースを共通化したまま、AI アプリを後から追加していける拡張モデル** が成立しています。

## 4. ライセンスと公開ポリシー

### 4-1. ライセンス

| 対象 | ライセンス | 備考 |
|---|---|---|
| ソースコード（大半） | **MIT License** | 商用利用可 |
| 一部ファイル（AWS Prototyping Program 由来） | Amazon Software License (ASL) | 該当ファイルは `THIRD-PARTY-NOTICES.txt` で明示 |
| ドキュメント | **CC BY 4.0**（Creative Commons Attribution） | 出典明示で再利用可 |

### 4-2. 公開ポリシーの特徴

- **Pull Request は受け付けない**
- **Issue は致命的な問題のみ受付**（データ損失・サービス障害・法令違反・重大なアクセシビリティ障壁）
- 機能追加要望・軽微なバグ・スタイル指摘はクローズ対象
- セキュリティ脆弱性は GitHub Security Advisory 経由

これは **「OSS化＝コミュニティ主導開発」ではなく、「政府が運用するソースコードの公開・参照可能化」** を主目的とする方針を明確化したものです。フォークして独自運用する想定で、本家にコントリビュートする運用は前提としていません。

## 5. ベース技術 — AWS GenU との関係

源内は **AWS が公開している OSS [Generative AI Use Cases (GenU)](https://github.com/aws-samples/generative-ai-use-cases)** をベースに、行政向けの拡張を加えて構築されています。

| 項目 | GenU（オリジナル） | 源内 |
|---|---|---|
| **対象ユーザー** | AWS 利用企業全般 | 政府・自治体・行政組織 |
| **デザイン** | AWS 標準 UI | **デジタル庁デザインシステム適用** |
| **チーム管理** | なし／簡易 | **多階層チーム管理機能** 追加 |
| **AIアプリ管理** | コード内 hard-code | **GUI からの登録・実行** に拡張 |
| **外部マイクロサービス** | 単一構成 | **ExApp プロトコル**で外部 AI サービスを差し込み可能 |
| **アクセシビリティ** | 一般水準 | **WCAG 2.0 AA 相当** での試験実施 |
| **運用機能** | 基本のみ | **監視・モニタリング・監査ログ** を追加 |
| **多言語** | 英語中心 | **日本語・英語** 両対応 |

GenU との関係は **「フォーク」ではなく「独立した開発」** とリポジトリ README に明記されています。設計思想を継承しつつコードベースは大幅に書き換えられており、GenU の更新を自動的に取り込む構造ではありません。

## 6. 公開後の社会的反応

### 6-1. GitHub 上の反応（2026年5月時点）

| 項目 | genai-web | genai-ai-api |
|---|---|---|
| Stars | 465 | 288 |
| Forks | 30 | 26 |
| 主要言語 | TypeScript 99.1% | Python 主体 |
| 最新リリース | v1.0.5（2026-04-30） | v1.0.3（2026-04-24） |
| 初回コミット | 2026-01-19（公開前準備期間） | 2026-01-27 |

### 6-2. 注目されている理由

- **「行政が実運用しているコードがそのまま読める」** という前例の少なさ
- **マルチクラウド対応**（AWS / Azure / Google Cloud）の参考実装としての価値
- **デジタル庁デザインシステム**が組み込まれた状態で公開されていること（民間サービスでも参考になる）
- **アクセシビリティ準拠**（WCAG 2.0 AA 相当）の実装事例として貴重

## 7. 源内が向くケース・向かないケース

### 7-1. 向くケース

- 自治体・大企業で、**外部 SaaS にデータを出さずに内製 AI 基盤を立てたい**
- 既に AWS / Azure / Google Cloud いずれかで運用基盤がある
- **複数チーム・複数業務にまたがる AI ポータル** を一元提供したい
- 行政・公共系で **アクセシビリティと監査ログ** が必要

### 7-2. 向かないケース

- **個人開発者**（ガバナンス機能が過剰）
- **クラウドベンダーロックを避けたい**（AWS CDK / Azure Bicep に依存）
- **本家への機能リクエスト** を期待する（PR 受付なし）
- **常に最新の LLM を試したい**（Bedrock 経由のモデル選択肢に依存）
