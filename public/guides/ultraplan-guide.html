1Ultraplan とは

従来の Claude Code では、/plan を実行するとターミナルがロックされ、計画が完成するまで別作業ができなかった。Ultraplan はプランニングのフェーズをAnthropicのクラウド（CCR）に移すことで、この問題を解決する。

ターミナルが自由

計画はクラウドで生成される。ローカルは3秒ごとのポーリングのみで、別ブランチの作業や複数セッションの並行実行が可能。

ブラウザでリッチなレビュー

インラインコメント・絵文字リアクション・アウトラインサイドバーを使い、計画の特定箇所を指定して修正依頼できる。

実行先を選択

承認後に「クラウドで実行してPRを作成」か「ターミナルに計画を戻して実行」かを選べる。

Opus 4.6 × 最大30分

クラウドセッションは最大30分のコンピューティング時間を確保し、大規模なアーキテクチャ分析にも対応する。

起動方法（3通り）

# 方法A: スラッシュコマンド（確認ダイアログあり）
/ultraplan migrate the auth service from sessions to JWTs

# 方法B: プロンプト内にキーワード（確認ダイアログあり）
I need an ultraplan for refactoring the payment module

# 方法C: ローカルプランから昇格（確認なし）
# /plan 実行 → 承認ダイアログで「Refine with Ultraplan」を選択
Remote Control と Ultraplan は同時に利用できない。どちらも claude.ai/code インターフェースを使うため、Ultraplan 起動時に Remote Control は切断される。

3GitHub 連携の仕組み

スナップショット取得（起動時）

Ultraplan を起動すると、その時点のリポジトリ状態が CCR に同期される。これは 点時間のコピー であり、起動後にローカルで加えた変更はクラウドセッションには反映されない。コミット＆プッシュ後に起動するのがベストプラクティス。

注意： Ultraplan は origin という名前のリモートからGitHub URLを探す。Bitbucket を origin にしてGitHubをセカンダリリモートにしている場合、認証エラーになる既知の不具合がある（回避策：GitHubリモートを origin にリネームする）。

セキュアプロキシ（git操作の仲介）

クラウドセッション内のgitクライアントは、実際のGitHubトークンを持たない。代わりに スコープ付き一時認証情報 を使ってAnthropicのセキュアプロキシに接続し、プロキシが検証・変換してGitHubへ転送する。

認証情報の隔離

GitHubトークンはVM内に入らない。漏洩リスクをゼロにする設計。

操作の制限

指定ブランチへの push のみ許可。clone・fetch・PR作成は可能。意図しないブランチへの書き込みを防止。

監査ログ

アウトバウンド通信はすべてプロキシ経由。完全に記録される。

ネットワーク制限

デフォルトは許可ドメインリストに限定。プロキシは GitHub 操作専用で、ネットワークアクセス設定とは独立している。

利用条件（GitHub）

  • GitHub リポジトリが必須（GitLab・Bitbucket・ローカルのみのリポジトリは不可）
  • Claude GitHub App をリポジトリにインストール済みであること
  • Amazon Bedrock・Google Cloud Vertex AI・Microsoft Foundry では利用不可
  • Team / Enterprise プランは Self-hosted GitHub Enterprise Server もサポート

6使い分けとベストプラクティス

Ultraplan を使うべきケース

大規模移行・リファクタリング

複数ファイルにまたがる変更や、アーキテクチャ上の決定が伴う作業。計画に30分かけることで実装のやり直しを防ぐ。

計画レビューを共有したい

ブラウザURLを共有すれば、チームメンバーが計画をレビューできる。

ローカルプランを使うべきケース

小さな変更・素早い反復

ローカルプランはターミナル上で即座にフィードバックできる。単純なタスクにUltraplanのオーバーヘッドは不要。

カスタムツール依存

ローカルのMCPサーバーやカスタムツールが必要な場合、クラウドセッションからはアクセスできない。

ベストプラクティス

  • Ultraplan 起動前に必ず git commit && git push する。コミット前の変更はスナップショットに含まれない可能性がある。
  • まずローカルプランで方向性を確認し、「Refine with Ultraplan」で昇格させると精度が高くなる。
  • プロジェクトルートに CLAUDE.md を置き、スタック・規約・触ってはいけないファイルを記述しておく。
  • タスク説明は具体的に。「auth をリファクタリング」より「/api/auth のセッション認証をJWTに置き換え、エンドポイントシグネチャを維持する」のほうが精度が上がる。
