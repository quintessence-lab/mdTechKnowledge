Self-Attention vs RNN vs CNN — 計算量比較（論文 Table 1）
n = 系列長, d = 表現次元, k = カーネルサイズ, r = 制限窓サイズ
層タイプ 層あたりの計算量 逐次操作数 最大パス長
Self-Attention O(n² · d) O(1) O(1)
Recurrent (RNN) O(n · d²) O(n) O(n)
Convolutional (CNN) O(k · n · d²) O(1) O(logk(n))
Self-Attention (restricted) O(r · n · d) O(1) O(n/r)
読み方のポイント:
• 逐次操作数 O(1) = 全位置を並列処理可能 → GPU/TPUの性能をフル活用
• 最大パス長 O(1) = 任意の2トークンが直接接続 → 長距離依存の学習が容易
• O(n) = 系列長に比例 → 長い系列ほど不利

トレードオフ: Self-Attentionの計算量 O(n²·d) は系列長の二乗に比例するため、n < d の一般的なケース（文の長さ < 512）では有利ですが、n ≫ d の超長系列ではRNNの O(n·d²) の方が効率的です。この問題は後のSparse Attention、FlashAttention等で改善されています。
